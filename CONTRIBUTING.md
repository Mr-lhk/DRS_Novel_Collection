# 仓库结构与投稿格式说明

此文档用于规范本项目的 GitHub 仓库目录结构，并统一投稿作品的文件格式、命名方式与相关要求，便于后续维护、整理与印刷。

## 仓库结构

以下为本项目的仓库目录结构：

```bash
DRS_Novel_Collection/
├── collected/                # 已收录作品（进入文选）
│   ├── Work_Title/           # 作品名称
│   │   ├── chapters/         # 章节文件夹（可选）
│   │   │   ├── assets/       # 插图、附加图像等（若不分章节，可直接位于作品文件夹下）
│   │   │   └── chap1.tex     # 正文
│   │   ├── metadata.json     # 元数据
│   │   └── main.tex          # 正文
│   └── Another_Work_Title/
│       └── ...
├── non_collected/            # 未收录作品（仅整理）
│   └── ...
├── Anthology.tex             # 文选
├── CONTRIBUTING.md           # 投稿指南（本文件）
└── README.md                 # 项目说明文档
```

说明：

``collected/`` 内的作品将进入文选，有机会被印刷。

``non_collected/`` 内的作品仅整理、不进入印刷。

每篇作品应有独立文件夹，存放正文与附属材料。

## 投稿格式

为便于统一排版与后续印刷，请尽量遵守以下规范：

### 文件格式

所有投稿作品必须使用 LaTeX（.tex）格式提交。

### 命名规范

对于文件名称及文件夹名称，均需使用英文命名

对于中文内容，优先使用翻译名，特殊情况下可使用拼音代替

对于多单词内容，每个单词的第一个字母大写，且单词间使用下划线连接

对于单个单词内容，全部使用小写字母

### 作品元数据（metadata）

在每篇作品文件夹内附带一份 metadata.json，示例：

```json
{
  "title": "作品标题",
  "author": "作者名称或化名",
  "anonymous": false,
  "tags": ["同人", "原创", "短篇"],
  "description": "一句话简介（可选）",
  "date": "2025-11-16",
  "permission": "CC BY-NC-ND 4.0"
}
```
